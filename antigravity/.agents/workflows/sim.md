# Sim Deep (Antigravity 버전)

// turbo-all

이 워크플로우는 코드가 정상 작동함을 넘어, **미래에 발생할 수 있는 잠재적 결함**을 단계별 시나리오로 검증합니다.

## 실행 절차

### 1단계: S1 - 실패 시뮬레이션 (Fail Sim)
- "3개월 후 + 규모 10배 확장" 상황에서 현재 코드가 어떻게 무너지는지 구체적인 실패 시나리오를 생성합니다.
- 데이터 병목, 메모리 누수, 의존성 충돌 등 가장 치명적인 지점을 식별합니다.

### 2단계: S2 - 근본 원인 탐색 (Root Cause)
- S1에서 발생한 실패의 근본적인 원인(RCA)을 추적합니다.
- 단순한 코드 수정을 넘어 아키텍처적 결함이나 확장성 한계를 분석합니다.

### 3단계: S3 - 수정 및 검증 (Verify)
- 분석된 원인을 해결하기 위한 수정 전략을 수립하고 코드를 보완합니다.
- 수정된 코드가 "3개월 후 + 10배 규모"에서도 견디는지 최종 시뮬레이션을 수행합니다.

## 사용법
- 3개 파일 이상 작업 후 자동 제안되거나, 사용자가 `/sim` 명령을 내릴 때 실행됩니다.
- 결과물은 [대상/기간/시나리오/판단(OK, WARN, FAIL)] 형태의 테이블로 제공합니다.
