# EazyCheck v6.1

> LOW = 바로 코딩. MID+ = 반드시 docs/eazycheck-detail.md Read 후 진행.

---

## 위험도 판단 (매 작업 필수)

변경 파일을 전부 나열한 후 아래 채점:

| 요소 | 조건 | 점수 |
|------|------|------|
| 파일 수 | 3-5: +1, 6-9: +2, 10+: +3 | 0-3 |
| 신규 의존성 | 새 라이브러리/패키지 | +1 |
| DB/스키마 | 마이그레이션, 테이블 변경 | +2 |
| 인증/보안 | auth, 권한, 토큰 | +2 |
| API 계약 | 엔드포인트 추가/변경 | +1 |
| 외부 연동 | 3rd party API, webhook | +1 |
| 신규 사용자 기능 | 새 UI, 새 워크플로우 | +1 |
| 기존 코드 삭제/교체 | 파일 삭제, 함수 교체 | +1 |

**0-1 = LOW / 2-3 = MID / 4+ = HIGH**

---

## 분기

```
LOW (0-1):  역추적(버그시) → 코드 → 완료
MID (2-3):  Read docs/eazycheck-detail.md → 리서치 → DMAD 1R → 구현명세 → 코드 → 자기검증 → sim 1회
HIGH (4+):  Read docs/eazycheck-detail.md → 리서치 → DMAD 2R → 구현명세 → 코드 → 자기검증 → sim 루프 2회
```

---

## 역추적 (버그 키워드 감지 시)

트리거: `에러, 오류, 안되, 문제, 실패, 버그, 500, 404, crash, fail`

```
[역추적]
증상: {1줄}
경로: {입력 → 거쳐가는 곳 → 터지는 위치}
수정 위치: {근본 원인 위치 확정}
```

1-5 파일에 적용. 6+ 파일은 sim S2가 커버.

---

## DMAD 핵심 규칙 (상세: docs/eazycheck-detail.md)

- 토론 전 변경 대상 파일 전부 Read 필수
- `파일명:라인: 코드내용` 인용 필수, 인용 없는 주장 금지
- 전부 수긍 금지, 반례 1개 이상

## smartLoop (실패 시)

| 실패 유형 | 재시작 지점 |
|----------|------------|
| lint 오류 | 직접 수정만 |
| 빌드 실패 | 역추적만 |
| sim FAIL | sim S2부터 |
| 설계 결함 | 전체 재시작 |

최대 3회. 3회 후 사용자 보고.
